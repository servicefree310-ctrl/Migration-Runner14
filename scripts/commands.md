# Zebvix — VPS Command Reference

All commands assume you're in `/opt/zebvix` and `DATABASE_URL` is exported.

---

## 1. First-time VPS Setup (Ubuntu 22.04)

```bash
# Clone repo
git clone <your-repo-url> /opt/zebvix
cd /opt/zebvix

# Set DB password
export ZEBVIX_DB_PASSWORD="YourSecurePassword123!"

# Full automated setup (Node, PostgreSQL, migrations, seeds, PM2)
bash scripts/vps-setup.sh
```

## 3. Seed Database

```bash
export DATABASE_URL="postgresql://zebvix:password@localhost:5432/zebvix"

# All seeds in dependency order (recommended)
bash scripts/vps-seed.sh

# OR individual seeds:
```

## 4. Build Frontends

```bash
cd /opt/zebvix

# Admin panel
PORT=23744 BASE_PATH=/admin/ pnpm --filter @workspace/admin run build
# Output: artifacts/admin/dist/

# User portal
PORT=23475 BASE_PATH=/user/ pnpm --filter @workspace/user-portal run build
# Output: artifacts/user-portal/dist/

# API server
pnpm --filter @workspace/api-server run build
# Output: artifacts/api-server/dist/index.mjs
```

## 5. PM2 — Process Manager

```bash
# Start all services (after first build)
pm2 start ecosystem.config.cjs

# View status
pm2 list

# View logs
pm2 logs zebvix-api
pm2 logs zebvix-api --lines 100
pm2 logs zebvix-api --err

# Restart after code update
pnpm --filter @workspace/api-server run build
pm2 restart zebvix-api

# Stop / delete
pm2 stop zebvix-api
pm2 delete zebvix-api

# Save process list (survives reboot)
pm2 save
pm2 startup    # follow the printed command to enable boot start
```

**ecosystem.config.cjs** (auto-generated by vps-setup.sh):

```js
module.exports = {
  apps: [{
    name: "zebvix-api",
    script: "./artifacts/api-server/dist/index.mjs",
    instances: "max",          // one per CPU core
    exec_mode: "cluster",
    env: { NODE_ENV: "production", PORT: "8080" },
    env_file: ".env",
    max_memory_restart: "1G",
  }]
};
```

## 6. Nginx

```bash
# Install config
sudo cp scripts/nginx-zebvix.conf /etc/nginx/sites-available/zebvix
sudo ln -sf /etc/nginx/sites-available/zebvix /etc/nginx/sites-enabled/zebvix

# Edit domain name
sudo sed -i 's/yourdomain.com/exchange.example.com/g' /etc/nginx/sites-available/zebvix

# Test and reload
sudo nginx -t
sudo systemctl reload nginx

# SSL certificate (free, via Let's Encrypt)
sudo certbot --nginx -d exchange.example.com -d www.exchange.example.com
sudo systemctl reload nginx

# Check logs
sudo tail -f /var/log/nginx/zebvix-access.log
sudo tail -f /var/log/nginx/zebvix-error.log
```

**Traffic routing:**

| URL | Serves |
|-----|--------|
| `https://yourdomain.com/api/*` | Express API (PM2 cluster, port 8080) |
| `https://yourdomain.com/api/ws` | WebSocket (price feeds, order book) |
| `https://yourdomain.com/admin/*` | Admin Panel (static `artifacts/admin/dist/`) |
| `https://yourdomain.com/user/*` | User Portal (static `artifacts/user-portal/dist/`) |
| `https://yourdomain.com/` | Redirects → `/user/` |
| `https://yourdomain.com/healthz` | Health check (no rate limit) |

## 7. Dev Server (Replit / local)

```bash

pnpm --filter @workspace/api-server run dev                      # port 8080

PORT=23744 BASE_PATH=/admin/ pnpm --filter @workspace/admin run dev

PORT=23475 BASE_PATH=/user/ pnpm --filter @workspace/user-portal run dev

# Typecheck everything
pnpm run typecheck

pnpm --filter @workspace/db run push
```

## 8. Update Deployed App (zero-downtime)

```bash
cd /opt/zebvix

# Pull latest code
git pull origin main

# Install new deps (if any)
pnpm install --frozen-lockfile

# Run new migrations (if any)
bash scripts/vps-migrate.sh

# Build API server
pnpm --filter @workspace/api-server run build

# Reload API (PM2 cluster — zero downtime)
pm2 reload zebvix-api

# Rebuild frontends if changed
PORT=23744 BASE_PATH=/admin/ pnpm --filter @workspace/admin run build
PORT=23475 BASE_PATH=/user/ pnpm --filter @workspace/user-portal run build
# nginx serves static files from dist/ — no reload needed
```

## 9. Useful psql Commands

```bash
# Connect to DB
psql "$DATABASE_URL"

# Check tables
\dt

# Check users
SELECT id, email, role, created_at FROM users ORDER BY id LIMIT 10;

# Check coins
SELECT symbol, name, current_price FROM coins ORDER BY market_cap_rank LIMIT 20;

# Check pairs
SELECT symbol, trading_enabled, futures_enabled FROM pairs ORDER BY id LIMIT 20;

# Check wallets
SELECT user_id, coin_id, balance FROM wallets WHERE balance > 0 LIMIT 20;
```

## 10. Firewall (ufw)

```bash
sudo ufw allow OpenSSH
sudo ufw allow 'Nginx Full'    # ports 80 + 443
sudo ufw deny 8080             # block direct API access — only via Nginx
sudo ufw deny 5432             # block direct DB access
sudo ufw enable
sudo ufw status
```
